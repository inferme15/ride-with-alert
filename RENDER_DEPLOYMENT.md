# 🚀 Render Deployment Guide for RideWithAlert

## 🔧 **Render Configuration**

### **1. Web Service Settings**
```
Name: ridewithAlert
Environment: Node
Build Command: npm ci && npm run build
Start Command: npm start
Health Check Path: /health
```

### **2. Environment Variables**
Set these in your Render dashboard:

**Required:**
- `NODE_ENV` = `production`
- `DATABASE_URL` = (Auto-generated by Render PostgreSQL)

**Optional (for SMS functionality):**
- `FAST2SMS_API_KEY` = Your Fast2SMS API key
- `POLICE_PHONE` = `100` (or your local emergency number)
- `HOSPITAL_PHONE` = `108` (or your local emergency number)

### **3. Database Setup**
- Create a PostgreSQL database in Render
- Name: `ridewithAlert-db`
- Plan: Free
- The `DATABASE_URL` will be automatically linked

## 🛠️ **Build Process**

The build process does the following:
1. **Install dependencies**: `npm ci`
2. **Build client**: `npm run build:client` → Creates React app in `dist/public/`
3. **Build server**: `npm run build:server` → Creates Node.js app as `dist/index.cjs`
4. **Start server**: `npm start` → Runs `node dist/index.cjs`

## 📁 **File Structure After Build**
```
dist/
├── index.cjs          # Server bundle
└── public/            # Client files
    ├── index.html     # React app entry
    ├── assets/        # CSS, JS bundles
    ├── manifest.json  # PWA manifest
    └── sw.js          # Service worker
```

## 🔍 **Troubleshooting**

### **Issue: Blank Page**
**Symptoms**: White/blank page when visiting your Render URL

**Solutions**:
1. **Check Build Logs**: Ensure both client and server build successfully
2. **Check Runtime Logs**: Look for static file serving errors
3. **Verify Environment Variables**: Ensure `NODE_ENV=production` is set
4. **Database Connection**: Check if `DATABASE_URL` is properly set

### **Issue: 500 Internal Server Error**
**Symptoms**: Server error when loading the page

**Solutions**:
1. **Check Database**: Ensure PostgreSQL database is connected
2. **Check Environment Variables**: Verify all required vars are set
3. **Check Logs**: Look for specific error messages in Render logs

### **Issue: API Endpoints Not Working**
**Symptoms**: Frontend loads but API calls fail

**Solutions**:
1. **Check Database Schema**: Run `npm run db:push` locally first
2. **Check CORS**: Ensure Socket.IO and API endpoints are properly configured
3. **Check Environment**: Verify production environment variables

## 🚀 **Deployment Steps**

### **Method 1: Automatic Deployment (Recommended)**
1. **Connect GitHub**: Link your GitHub repository to Render
2. **Create Web Service**: Use the settings above
3. **Add Database**: Create PostgreSQL database and link it
4. **Set Environment Variables**: Add required environment variables
5. **Deploy**: Render will automatically build and deploy

### **Method 2: Manual Deployment**
1. **Build Locally**: Run `npm run build`
2. **Test Locally**: Run `npm start` to test production build
3. **Push to Git**: Commit and push your changes
4. **Deploy on Render**: Trigger manual deployment

## 🐛 **Common Issues & Fixes**

### **Build Fails**
- Check `package.json` scripts are correct
- Ensure all dependencies are in `dependencies` (not `devDependencies`)
- Verify Node.js version compatibility

### **Database Connection Fails**
- Ensure PostgreSQL database is created and linked
- Check `DATABASE_URL` environment variable
- Verify database schema is pushed

### **Static Files Not Loading**
- Check build output in `dist/public/`
- Verify static file serving configuration
- Check for CORS issues
